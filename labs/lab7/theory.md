## Теория

### Динамическое программирование (DP)

**Динамическое программирование** — метод решения задач путём разбиения на перекрывающиеся подзадачи и сохранения их результатов, чтобы не вычислять одно и то же дважды.

Два признака задачи, решаемой через DP:
- **Оптимальная подструктура** — оптимальное решение задачи строится из оптимальных решений подзадач
- **Перекрывающиеся подзадачи** — одни и те же подзадачи встречаются многократно

Два подхода:
- **Top-down (мемоизация)** — рекурсия + кэш уже вычисленных значений
- **Bottom-up (табуляция)** — итеративное заполнение таблицы от базового случая к ответу

---

### Жадные алгоритмы

**Жадный алгоритм** — на каждом шаге выбирает локально оптимальное решение, рассчитывая получить глобально оптимальный результат.

Отличие от DP: жадный не пересматривает уже сделанные выборы. Работает не для всех задач, но там где работает — даёт O(n) вместо O(n²).

Признак применимости: **свойство жадного выбора** — локально лучший выбор всегда входит в глобально оптимальное решение.

### #198 — House Robber

**Условие:** вдоль улицы стоят дома с деньгами `nums[i]`. Нельзя грабить два соседних дома. Найти максимальную сумму.

**Идея:** dp[i] = максимум из двух вариантов:
- Не грабим дом `i` → берём dp[i-1]
- Грабим дом `i` → берём dp[i-2] + nums[i]

```
nums = [2, 7, 9, 3, 1]

i=0: prev2=0, prev1=2
i=1: prev2=2, prev1=max(2, 0+7)=7
i=2: prev2=7, prev1=max(7, 2+9)=11
i=3: prev2=11, prev1=max(11, 7+3)=11
i=4: prev2=11, prev1=max(11, 11+1)=12  →  ответ: 12
```

**Оптимизация памяти:** вместо массива dp[n] достаточно двух переменных `prev1` и `prev2`, так как dp[i] зависит только от двух предыдущих значений.

**Сложность:** O(n) по времени, **O(1)** по памяти.

```python
def rob(self, nums):
    prev2, prev1 = 0, 0
    for num in nums:
        prev2, prev1 = prev1, max(prev1, prev2 + num)
    return prev1
```

### #322 — Coin Change

**Условие:** даны монеты `coins` и сумма `amount`. Найти минимальное количество монет для составления суммы (или -1, если невозможно).

**Идея:** bottom-up DP. `dp[i]` = минимальное число монет для суммы `i`.
- База: `dp[0] = 0`
- Переход: `dp[i] = min(dp[i - coin] + 1)` для каждой монеты `coin ≤ i`

```
coins = [1, 5, 10, 25], amount = 36

dp[0]  = 0
dp[1]  = dp[0] + 1 = 1     (монета 1)
dp[5]  = dp[0] + 1 = 1     (монета 5)
dp[10] = dp[0] + 1 = 1     (монета 10)
dp[25] = dp[0] + 1 = 1     (монета 25)
dp[36] = dp[11] + 1 = dp[1] + 1 + 1 = 3   (25+10+1)
```

**Ключевой момент:** инициализируем `dp[i] = inf` (недостижимо), чтобы корректно брать минимум. Если в конце `dp[amount] == inf` — решения нет.

**Сложность:** O(amount × len(coins)) по времени, O(amount) по памяти.

```python
def coinChange(self, coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for i in range(1, amount + 1):
        for coin in coins:
            if coin <= i:
                dp[i] = min(dp[i], dp[i - coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1
```

### #300 — Longest Increasing Subsequence

**Условие:** найти длину наибольшей строго возрастающей подпоследовательности (LIS).

**Идея (O(n log n) — метод Пейшенс):** поддерживаем массив `tails`, где `tails[i]` — наименьший хвостовой элемент всех возрастающих подпоследовательностей длины `i+1`.

Для каждого нового числа:
- Если оно больше всех элементов `tails` — расширяем массив (LIS стал длиннее)
- Иначе — заменяем первый элемент `tails`, который ≥ числа (бинарный поиск)

```
nums = [10, 9, 2, 5, 3, 7, 101, 18]

10  →  tails = [10]
9   →  tails = [9]           (заменяем 10)
2   →  tails = [2]           (заменяем 9)
5   →  tails = [2, 5]        (добавляем)
3   →  tails = [2, 3]        (заменяем 5)
7   →  tails = [2, 3, 7]     (добавляем)
101 →  tails = [2, 3, 7, 101](добавляем)
18  →  tails = [2, 3, 7, 18] (заменяем 101)

len(tails) = 4  →  ответ: 4
```

**Ключевой момент:** `tails` — не реальная LIS, а вспомогательная структура. Её длина = длина LIS, но конкретные элементы могут не образовывать реальную подпоследовательность.

**Сложность:** O(n log n) по времени, O(n) по памяти.

```python
def lengthOfLIS(self, nums):
    tails = []
    for num in nums:
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)
        else:
            tails[pos] = num
    return len(tails)
```

### #134 — Gas Station

**Условие:** по кругу расставлены заправки. На станции `i` получаем `gas[i]` бензина и тратим `cost[i]` на переезд к следующей. Найти стартовую станцию, с которой можно объехать круг (или -1).

**Идея (жадный):** два ключевых наблюдения:

1. Если `sum(gas) < sum(cost)` — решения не существует совсем
2. Если решение существует — оно единственно. Найти его можно за один проход:

Идём по станциям, накапливая остаток `tank`. Если `tank < 0` — с текущего старта добраться невозможно, сдвигаем старт на следующую станцию и обнуляем `tank`.

```
gas  = [1, 2, 3, 4, 5]
cost = [3, 4, 5, 1, 2]
diff = [-2,-2,-2, 3, 3]

start=0, tank=-2 < 0 → start=1, tank=0
start=1, tank=-2 < 0 → start=2, tank=0
start=2, tank=-2 < 0 → start=3, tank=0
i=3: tank=3
i=4: tank=6

ответ: start=3 ✓
```

**Почему это работает:** если на отрезке [A, B] у нас заканчивается топливо, то ни одна станция из [A, B] не может быть стартом — иначе они бы не набрали суммарный дефицит этого отрезка.

**Сложность:** O(n) по времени, O(1) по памяти.

```python
def canCompleteCircuit(self, gas, cost):
    if sum(gas) < sum(cost):
        return -1
    start, tank = 0, 0
    for i in range(len(gas)):
        tank += gas[i] - cost[i]
        if tank < 0:
            start = i + 1
            tank = 0
    return start
```
