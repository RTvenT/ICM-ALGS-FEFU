# Лабораторная работа №5 — Бинарные деревья

## Теория

### Бинарное дерево

**Бинарное дерево** — иерархическая структура данных, в которой каждый узел имеет не более двух потомков: **левый** и **правый**.

```
        4
       / \
      2   7
     / \ / \
    1  3 6  9
```

Ключевые понятия:
- **Корень (root)** — верхний узел без родителя
- **Лист (leaf)** — узел без потомков
- **Высота дерева** — длина наибольшего пути от корня до листа
- **Поддерево** — любой узел вместе со всеми его потомками

---

### Обходы дерева

| Обход | Порядок | Применение |
|---|---|---|
| Preorder (pre) | корень → левое → правое | копирование дерева |
| Inorder (in) | левое → корень → правое | отсортированный вывод BST |
| Postorder (post) | левое → правое → корень | удаление дерева |
| Level-order (BFS) | уровень за уровнем | поиск кратчайшего пути |

---

### Бинарное дерево поиска (BST)

**BST** — бинарное дерево с дополнительным свойством:

> Для каждого узла `N`: все значения в **левом** поддереве **строго меньше** `N.val`, все значения в **правом** поддереве **строго больше** `N.val`.

```
        6
       / \
      2   8
     / \ / \
    0  4 7  9
      / \
     3   5
```

Благодаря этому свойству поиск, вставка и удаление работают за **O(h)**, где h — высота дерева. В сбалансированном дереве h = O(log n).

---

### DFS vs BFS на деревьях

- **DFS (рекурсия / стек)** — уходит вглубь ветки до листа, потом возвращается. Подходит для задач, где нужно обработать всё дерево.
- **BFS (очередь)** — обходит узлы уровень за уровнем. Подходит для задач с понятием "уровня" или "расстояния от корня".

### #226 — Invert Binary Tree

**Идея:** рекурсивный DFS. На каждом узле меняем левого и правого потомка местами, затем рекурсивно делаем то же самое для обоих поддеревьев.

```
    4          4
   / \   →    / \
  2   7      7   2
 / \ / \    / \ / \
1  3 6  9  9  6 3  1
```

**Сложность:** O(n) по времени, O(h) по памяти (стек рекурсии).

**Ключевой момент:** порядок рекурсии не важен — можно менять потомков до или после рекурсивных вызовов, результат одинаков.

```python
def invertTree(self, root):
    if not root:
        return None
    root.left, root.right = root.right, root.left
    self.invertTree(root.left)
    self.invertTree(root.right)
    return root
```

### #104 — Maximum Depth of Binary Tree

**Идея:** глубина дерева = 1 + максимум из глубин левого и правого поддеревьев. Базовый случай: пустой узел имеет глубину 0.

```
    3          depth = 3
   / \
  9  20        depth = 2
    /  \
   15   7      depth = 1
```

**Сложность:** O(n) по времени, O(h) по памяти.

**Ключевой момент:** задача сводится к одной строке после базового случая — это классический пример, где рекурсия делает всё за нас.

```python
def maxDepth(self, root):
    if not root:
        return 0
    return 1 + max(self.maxDepth(root.left), self.maxDepth(root.right))
```

### #102 — Binary Tree Level Order Traversal

**Идея:** BFS с очередью. На каждой итерации внешнего цикла обрабатываем ровно столько узлов, сколько их сейчас в очереди — это и есть один уровень.

```
    3       → [[3], [9, 20], [15, 7]]
   / \
  9  20
    /  \
   15   7
```

Ключевой приём: `for _ in range(len(queue))` фиксирует размер уровня **до** того, как мы начинаем добавлять потомков следующего уровня.

**Сложность:** O(n) по времени и памяти.

```python
def levelOrder(self, root):
    if not root:
        return []
    result = []
    queue = deque([root])
    while queue:
        level = []
        for _ in range(len(queue)):
            node = queue.popleft()
            level.append(node.val)
            if node.left: queue.append(node.left)
            if node.right: queue.append(node.right)
        result.append(level)
    return result
```

### #98 — Validate Binary Search Tree

**Идея:** наивная проверка «левый < текущий < правый» не работает для всего дерева. Нужно передавать **допустимый диапазон** `(lo, hi)` в каждый узел.

```
    5          # допустимо (-inf, +inf)
   / \
  1   4        # 4 допустимо (5, +inf)? НЕТ → False
     / \
    3   6
```

При переходе налево верхняя граница `hi` обновляется до значения текущего узла. При переходе направо обновляется нижняя граница `lo`.

**Сложность:** O(n) по времени, O(h) по памяти.

**Ключевой момент:** `float('-inf')` и `float('inf')` как начальные границы позволяют не делать специальный случай для корня.

```python
def isValidBST(self, root):
    def validate(node, lo, hi):
        if not node:
            return True
        if not (lo < node.val < hi):
            return False
        return validate(node.left, lo, node.val) and validate(node.right, node.val, hi)
    return validate(root, float('-inf'), float('inf'))
```

### #235 — Lowest Common Ancestor of BST

**LCA (Наименьший общий предок)** — самый глубокий узел, который является предком обоих узлов `p` и `q`.

**Идея:** используем свойство BST итеративно, без рекурсии и доп. памяти:
- Если оба `p` и `q` меньше текущего узла → LCA в левом поддереве
- Если оба больше → LCA в правом поддереве
- Иначе → текущий узел и есть LCA (один из них слева, другой справа, или один из них и есть текущий узел)

```
        6
       / \
      2   8
     / \ / \
    0  4 7  9
      / \
     3   5

LCA(2, 8) = 6   # расходятся в разные стороны от 6
LCA(2, 4) = 2   # 4 в правом поддереве 2 → 2 сам является LCA
```

**Сложность:** O(h) по времени, **O(1)** по памяти — итеративный подход не использует стек рекурсии.

```python
def lowestCommonAncestor(self, root, p, q):
    while root:
        if p.val < root.val and q.val < root.val:
            root = root.left
        elif p.val > root.val and q.val > root.val:
            root = root.right
        else:
            return root
```
